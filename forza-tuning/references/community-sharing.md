# Community Sharing

Use this when the user asks how to name, publish, describe, tag, share, or optimize a tune for downloads, use, search, ratings, or community trust. Treat tune publishing as a player-expectation problem: the title and description should tell the correct audience exactly what job the tune does.

## Source And Scope

- Default to Forza Horizon 6 when the user does not name a game.
- FH6 official progression/event source: Forza.net, `Meet the Three Pathways of Progression in Forza Horizon 6`, May 12 2026. It lists official Festival events as Road, Dirt, Cross Country Races, Time Attack Circuits, Drag Meets, PR Stunts, and Bonus Boards; Discover Japan includes Street and Touge Races; Horizon Play includes Touge Showdown, Spec Racing, Horizon Racing, Horizon Drift, Custom Racing, and Custom Drifting.
- Treat event labels as live-service-visible terms. If the user asks for current playlist or exact event restrictions, verify current in-game/official info before giving a final publishing recommendation.

## Publishing Goal

Maximize downloads and high ratings by matching one car, one class, one event family, and one driver expectation. A slightly slower tune that is honest, stable, and easy to use often rates better than an edgy tune with vague claims.

Prioritize:

1. Put the class and use case in the title.
2. Name restrictions honestly.
3. Avoid overclaiming if the tune is not leaderboard-tested.
4. Describe who the tune is for: Trial, seasonal, Rivals, cruise, road, touge, drag, drift, dirt, cross country, purist.
5. Include the share code, event, assists/input assumptions, and proof result when available.

## In-Game Tuning Keywords

Use this section whenever the user asks for `Keyword 1`, `Keyword 2`, `Tuning Keyword 1`, `Tuning Keyword 2`, or the tune description fields. These are in-game dropdown values, not free-text race labels. Do not invent values such as `Touge`, `Road`, `RWD`, `Purist`, `No Aero`, or `Stable` for Keyword 1 or Keyword 2 unless the user provides them as current in-game options.

Allowed `Tuning Keyword 1` values:

- `all around`
- `circuit`
- `drag`
- `drift`
- `Grip`
- `off the line`
- `top speed`

Allowed `Tuning Keyword 2` values:

- `average`
- `begineer`
- `Best for Handling`
- `best for speed`
- `Close-gear ratio shifting`
- `Expert`
- `Good All around`
- `Good for handling`
- `Good for speed`
- `Novice`
- `Wide-gear ratio shifting`

For technical road, mountain road, or touge-style tunes, usually choose `Grip` for Keyword 1 and `Best for Handling` or `Good for handling` for Keyword 2. Put `touge`, `RWD`, `purist`, `no swap`, `stock engine`, `no aero`, and route-specific details in the title or description instead of the in-game keyword fields.

For the in-game tune description field, assume a 64-character maximum unless the user provides a different limit. Count spaces and punctuation. Before returning a description, count it and keep it at or under the limit. Known exact-limit example, 64 characters: `A700 RWD mountain road build. Stock drivetrain, sharp front bite`

For beginner-friendly or casual controller tunes, use `Novice` or `begineer` only when the setup is forgiving, stable, and not shift-sensitive. Use `Expert` only for edgy, manual, leaderboard, drag-launch, or drift tunes that require precise inputs.

For gearing labels, use `Close-gear ratio shifting` when the tune uses short gears for technical corner exits or acceleration. Use `Wide-gear ratio shifting` when the tune intentionally uses longer gears for speed, traction management, or fewer shifts.

## Race And Event Keywords

Use these as title and description language unless they also appear in the in-game keyword list above.

| Keyword | Player expectation |
|---|---|
| Road | Paved all-round handling, braking, cornering, stable exits |
| Circuit | Repeatable lap balance, trail braking, direction changes, not only top speed |
| Sprint | More speed-biased road tune with enough grip and braking |
| Street | Paved/high-speed surprise-corner use; often less track-like than circuit |
| Time Attack | Sharp lap-time focus; may be less forgiving than casual tunes |
| Rivals | Repeatable solo pace, clean-lap consistency, leaderboard intent |
| Touge | Tight mountain-road response, braking, front bite, hairpin exit, rotation |
| Dirt | Mixed loose surfaces, gravel/forest paths, predictable slides |
| Rally | Dirt-biased performance with compliance, gearing, and recoverability |
| Cross Country or CC | Rough terrain, water/bumps, landings, high ride height, AWD stability |
| Drag | Launch, gearing, tire heat, trap speed; not for normal road races |
| Drift | Angle, transition, controllable wheelspin, score or tandem use |
| Trial | Playlist co-op win consistency for average players |
| Seasonal | Built around current playlist restrictions; include restriction if known |
| EventLab | Built for a custom route; include route style if known |
| Open | Online consistency, traffic/contact tolerance, broad route coverage |

## Build And Feel Keywords

| Keyword | Player expectation |
|---|---|
| Grip | Stable cornering and braking before top speed |
| Meta | Fast/common competitive build path; must be genuinely strong |
| Easy | Controller-friendly, stable, low drama |
| EZ Win | Seasonal/Trial-focused; should be forgiving more than edgy |
| Stable | Resists snap oversteer, braking instability, and rough-surface surprises |
| No Aero | No Forza adjustable aero; preserves look or top-end |
| Aero | Uses adjustable aero; expect more grip and possible drag cost |
| AWD | Easy launch, forgiving throttle, online consistency |
| RWD | Rotation, top-end efficiency, purist/skill appeal |
| FWD | Native/front-drive behavior; front grip and exit traction matter |
| Auto Friendly | Gearing works well with automatic shifting |
| Manual | Shift points matter; include if auto may be worse |
| Controller | Smooth enough for gamepad inputs |
| Wheel | Tuned/tested for wheel users; do not claim unless tested |
| Wet | Rain/wet-road-friendly; avoid edgy tire/ARB/alignment assumptions |
| Snow | Snow tire/surface intent; not a normal road-grip label |

## Purist Keywords

Purist is a separate audience, not just a tune adjective. Purist players search for trust and identity preservation; they often punish ambiguity more than casual users do.

| Keyword | Expected meaning |
|---|---|
| Purist | Usually no engine swap, no drivetrain swap, no Forza aero |
| Stock Motor or Stock Engine | Original engine retained; power upgrades may still be installed |
| No Swap | Usually no engine swap and/or no drivetrain swap; clarify which |
| Stock Drivetrain | Original FWD/RWD/AWD layout retained |
| No Aero or No Forza Aero | No adjustable Forza splitter/wing; clarify body-kit aero if relevant |
| OEM+ | Tasteful upgrade path, visually and mechanically close to stock |
| Street | Road-legal feel, not a full race-car conversion |
| Replica | Built to mimic a real-world spec, tuner car, race trim, or factory variant |
| Stock Looks | Factory-like visuals; may still include hidden performance upgrades |
| No Widebody | Keeps factory body shape |

When using `Purist`, state restrictions in the description. Do not use `Purist` if the build has an engine swap, drivetrain swap, Forza aero, or widebody unless the title/description clearly says the community ruleset allows it.

Good purist descriptions:

- `Stock engine, stock drivetrain, no Forza aero. A700 road/touge. Controller friendly, stable braking, clean exits.`
- `B600 OEM+ street build. No swap, no widebody, stock looks. Built for road sprints and casual rivals.`

## Title Patterns

Keep titles compact and searchable. Put class first, then job, then defining restriction or feel.

Use:

- `A700 Road Grip`
- `S1 Dirt Rally AWD`
- `B600 Touge RWD`
- `A700 Trial EZ`
- `S2 Time Attack`
- `C500 Purist No Aero`
- `B600 OEM+ Street`
- `S1 Stock Motor Grip`
- `Drag 1/4 Manual`
- `A700 No Swap Touge`

Avoid:

- `BEST TUNE` unless there is proof and the tune is genuinely competitive.
- `FAST` without event context.
- `GODLIKE`, `BROKEN`, or clickbait if the tune is merely stable.
- `Purist` when the build uses hidden swaps or Forza aero.

## Description Pattern

Use a short, evidence-first description when space allows:

`[Restrictions/build identity]. Built for [event/class]. [Input/assist assumption]. Strength: [main behavior]. Proof: [lap/time/win/difficulty if known]. Share code: [code].`

Examples:

- `AWD A700 Trial tune. Controller/auto friendly. Stable braking, clean exits, built for playlist co-op.`
- `RWD B600 touge. Stock engine/drivetrain, no Forza aero. Sharp entry, short gearing, tested on mountain sprints.`
- `S1 800 dirt rally. AWD, rally suspension, compliant landings, stable over bumps.`
- `S2 900 time attack. Grip/aero build for clean Rivals laps. Manual recommended.`

## Rating Strategy

Prefer honest fit over maximum hype.

- Publish early for Festival Playlist cars, reward cars, Trial restrictions, new meta cars, barn finds, and highly searched nostalgia cars.
- Make one tune per clear use case instead of one vague all-purpose tune when the car supports multiple audiences.
- For casual/playlist users, bias toward stability, braking, launch, and automatic gearing.
- For Rivals/time attack users, name the route or discipline and avoid claiming broad comfort if the tune is edgy.
- For drag, state distance or expected use: quarter mile, half mile, mile, runway, manual, auto, launch method if important.
- For drift, state score/angle/tandem/power style if known.
- Re-test after any build change that affects tire compound, tire width, track width, aero, drivetrain, power, suspension, gearing, or diff.

## Trust Checklist

Before recommending a publish title or description, check:

- Class ceiling and exact PI are clear.
- Event family is clear.
- Drivetrain and major swaps are clear.
- Purist/no-aero/no-swap claims are true.
- Auto/manual expectation is clear when gearing depends on shifting.
- The tune is not described as leaderboard/meta unless it has proof or the user asked for marketing language.
- The title does not imply a different event than the build actually supports.
