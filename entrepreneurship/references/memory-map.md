# Entrepreneurship Memory Map

Use this file first when the request is non-trivial. Load only the references needed for the current decision.

## Default Loading Path

- Stay in `SKILL.md` for quick, low-stakes questions that need only a direct recommendation.
- Read `problem-map.md` first when the answer depends on gathering decision-specific context.
- Read `pattern-index.md` next and load only the smallest relevant file or two from `patterns/`.
- Read a matching file from `founders/` only when the recommendation needs named founder context, a source-specific caveat, or a clear "copy this, not that" example.
- Read `decision-playbook.md` for any meaningful product, pricing, positioning, go-to-market, retention, or prioritization decision.
- Read `feedback-loop.md` when the answer feels generic, too broad, under-instrumented, or high-stakes.
- Read `interview-extraction-template.md` when adding new interviews to the corpus.
- Read `research-synthesis.md` when refining the skill, checking why the workflow is structured this way, or tightening the evaluation loop.

## Load Discipline

- Prefer one main reference at a time.
- Reach first for a pattern file, not a broad synthesis file.
- Add a founder card only if it clearly changes the recommendation.
- Do not load every file by default.
- Keep the runtime loop short: classify problem, gather the discriminating context, compare options, recommend, instrument.
- Use the research file only when improving the skill or debugging weak outputs.
- Use the legacy synthesis files only for maintenance, comparison, or gap-finding across patterns.
