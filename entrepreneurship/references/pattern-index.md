# Pattern Index

Use this file after `problem-map.md`. Its job is to help you choose the smallest useful subset of the interview corpus.

## How To Use This File

1. Identify the primary decision.
2. Pick 1-2 matching pattern files from `references/patterns/`.
3. Read a founder card only if the recommendation benefits from named source-specific nuance.
4. Do not load many patterns at once unless the decision is genuinely multi-part.

## Pattern Selection Rule

Choose the pattern that best matches:
- buyer behavior
- distribution channel
- cost structure
- stage of the business
- founder strengths and constraints

If no pattern clearly fits, fall back to `decision-playbook.md` and recommend a smaller test instead of forcing a founder analogy.

## Founder Card Rule

Reach for a founder card only when one of these is true:
- the user asks what a specific founder did
- the named founder move materially sharpens the recommendation
- the pattern carries a non-obvious precondition that is easier to explain through the founder's context

Default: pattern first, founder second.
