# Interview Extraction Template

Use this file when adding new interviews to the entrepreneurship skill. The goal is not to summarize everything. The goal is to extract reusable operator judgment.

## Extraction Workflow

1. Capture the founder and company context.
2. Extract only decision-relevant claims, moves, and constraints.
3. Convert repeated ideas into candidate patterns.
4. Write what made the move work there.
5. Write what would make it fail elsewhere.
6. Link the note to existing patterns or create a new pattern only if the move is genuinely reusable.

## Promotion Rule

Promote an interview insight into `references/patterns/*.md` only if:
- it solves a recurring startup decision
- it includes a clear fit condition
- it includes a clear misuse risk
- it would still be useful without the original transcript in context

Otherwise keep it as founder-specific nuance in a founder card.

## Pattern File Checklist

Each new pattern should answer:
- what decision it helps with
- when to use it
- what must be true for it to work
- what to copy
- what not to copy
- failure modes
- what to measure
- which founder cards support it
